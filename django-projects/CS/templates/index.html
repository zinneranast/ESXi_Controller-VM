    About project

    Abstract

    The main reason to start this project is desire to answer the question: how can we connect a storage resource to a client host by a virtual network automatically?

    The project deals with hypervisor technologies, virtual networking, SDN and modern Linux networking. The work was related to the use and study the following technologies, which allow to configure network automatically: OpenFlow-style SDN, NSX-style virtual networks, VxLAN, Open vSwitch.

    Reasons

    We have a storage in a data center.Owner of the storage is a provider and he wishes to provide services to his customers (NFS in our case, it uses RPC, see NAS Server Imitator and rpcbind, rpcbind).

    Customers (tenants) want to bring their own existing network infrastructure, which is divided on VLAN subnets. Tenants have to share array and own some part of it. We want multiple tenants to use it. Provider canâ€™t isolate tenants with VLAN technology (because quantity of VLAN is limited on 4096). The main question is how to provide connectivity between the array and tenant host?

    So, we widely used VxLAN technology to answer this. In fact, we tried to add VxLAN connectivity to an array by putting a VTEP (VxLAN Tunnel End Point) on it.

    Targets

    There are three main targets of this project:

    1. Understand principles and limitations of current Virtual Network (VN) technology. Explore VxLAN, Open vSwitch, SDN technologies and Linux kernel networking subsystem in network automated configuration.

    2. Create prototype/number of prototypes that will demonstrate how a virtual network could be configured to allow the multi-tenant server access.

    3. During the project development there appeared a task to make possible RPC access to the multi-tenant server.
